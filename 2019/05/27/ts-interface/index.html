接口

简介

Typescript的其中一个核心原则就是类型检查关注于检查值的外形(One of Typescript’s core principles is that type checking focuses on the shape that values have). 有时称为”鸭子类型”或”结构子类型”. Typescript中接口的角色就是为这些类型命名, 这在定义你自己代码或项目之外其他代码的协议是非常强大的.

上一个例子中的接口labeledValue是一个可以用于描述需求的名字. 它表述了具有一个单独的属性, 且类型是string, 名称是label. 值得注意的是和其他语言不通, 我们并没有显示的将传入参数实现该接口. 在这里, 只有外形是关键. 如果我们传入的参数符合列出的需求, 那么就是允许的.

一个应该指出的地方是类型检查器并没有要求属性要以一定的顺序排序, 只要求接口要求的属性存在且类型相符即可.

readonly vs const

一个记住readonly和const最简单的方法就是问下自己是使用在变量上海市属性上. 变量使用const, 同时属性使用readonly.

然而, Typescript对此的态度是在这段代码中可能隐含bug. 对象字面量被特殊对待且在赋值给一个变量或参数时受到额外的属性检查. 如果对象字面量在传入目标变量时有目标变量类型中未定义的属性, 将导致一个错误.
